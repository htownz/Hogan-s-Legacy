# Act Up Platform: Full Project Summary & Dev Notes

## Overview
**Act Up** is a 100% free, open-source civic engagement platform built to empower individuals to understand, engage with, and influence the legislative process. Initially focused on Texas (state level), it will later scale to city, county, and federal levels.

**Core Message**: _“We don’t editorialize. We reveal. We don’t judge. We expose. We don’t tell you what matters. We help you see what does—clearly and personally.”_

**Mission**: Make government transparency accessible and actionable for everyone—empowering communities to take control, demand accountability, and create meaningful change.

**Vision**: A future where informed, engaged communities reshape the relationship between government and the people.

---

## Key Features (Planned or In Progress)
- Full bill search and legislative database with real-time updates
- Personalized bill impact explanation AI ("If this bill passes, it means... for YOU")
- Track amendments, regulations, historical data, statutes, and rules
- Points of order and rules violations flagged in real-time
- Public and private user modes
- User-generated issue lists and community watch lists
- Notification and alert system for tracked items
- Scorecard alternatives (non-partisan, non-editorialized)
- Super User Program with roles:
  - **Catalyst** (knowledge drivers)
  - **Amplifier** (message spreaders)
  - **Convincer** (influencers of belief/action)

## Tech Stack (Planned/In Progress)
- **Frontend**: React (preferred)

- **Database**: PostgreSQL (likely)
- **API Gateway**: Alpaca (for trading alerts – optional feature)

- **Open-source delivery**: GitHub
- **Social deployment**: Instagram (initial traction + education delivery)

## Misc Notes
- No media-style commentary or political slant.
- No personal branding; platform identity is distinct.
- Inspired by Malcolm Gladwell’s Tipping Point, aiming for 25% engagement.
- Platform will grow with user-generated traction and community ownership.
- MVP should be up by January 2025 (Texas Legislative Session start).

## Repository Handoff Instructions
To continue:
- Clone or fork the repository and install dependencies for both backend and frontend
- Use `.env.example` to create environment variables for dev
- Review and test backend endpoints (Postman/Insomnia recommended)
- Set up PostgreSQL locally or via cloud (e.g., Supabase, Railway)
- Focus MVP on core loop: **search ➝ understand ➝ track ➝ act**
- Add proper testing, error handling, and minimal UX polish
- Optional: integrate PineScript trading alerts for politically driven financial plays (future)
