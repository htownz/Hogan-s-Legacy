# 1 Week / 2 Week / 3 Week / 1 Month Roadmap

Date: 2026-04-04

## Goal

Ship the first credible version of a `Policy Market Screen` and build the infrastructure that turns it from a smart dashboard into a real political market terminal.

## Week 1

Theme: `Make the flagship real with existing live data`

Outcome:

- a visible first version of the `Policy Market Screen` using currently live data sources

Scope:

1. Create the new front-end route and page shell.
2. Populate V1 from live-capable endpoints:
   - hearings
   - committee-intel sessions
   - committee-intel session detail
   - digest
   - alerts
   - issue rooms
   - intelligence briefing
3. Add the main V1 modules:
   - market movers rail
   - selected committee/hearing panel
   - action desk
   - catalyst tape
4. Add a Business & Commerce preset since that hearing/session is already a live proof point.
5. Clearly label model output as:
   - directional
   - evidence-backed
   - not full prediction odds yet

Must-have deliverable:

- a demoable screen that feels different on day one

## Week 2

Theme: `Fix trust and data quality`

Outcome:

- committee-intel output becomes more credible and less embarrassing

Scope:

1. Improve committee-intel speaker and entity resolution:
   - legislator alias maps
   - committee roster matching
   - witness roster alignment
   - phonetic cleanup
2. Fix obvious false entities like committee names being treated as people where possible.
3. Add quality flags to committee-intel entities:
   - matched
   - low confidence
   - unresolved
4. Add committee member pressure cards to the market screen.
5. Add evidence drill-down from summary -> key moment -> segment.

Must-have deliverable:

- Business & Commerce output that a serious lobbyist would not immediately laugh at

## Week 3

Theme: `Turn insight into advantage`

Outcome:

- the screen begins to behave like a desk tool instead of a reporting tool

Scope:

1. Add support depth and resistance depth scoring.
2. Add a catalyst model for hearings and bills:
   - scheduled catalysts
   - emerging catalysts
   - expired catalysts
3. Add action recommendation blocks:
   - best next move
   - who to contact
   - what to prepare
   - what evidence to cite
4. Add one-click operational actions from the screen:
   - open issue room
   - create task
   - add stakeholder note
   - generate hearing memo
   - generate client alert
5. Add saved views for firms:
   - defense
   - offense
   - committee day
   - client risk

Must-have deliverable:

- a user can move from signal to action without leaving the flagship screen

## Month 1

Theme: `Make the market model real`

Outcome:

- the product crosses from “smart workflow” into “market intelligence system”

Scope:

1. Fix missing schema/infrastructure for:
   - predictions
   - relationships
   - session lifecycle
2. Turn predictions on in a conservative form:
   - implied odds
   - confidence bands
   - trend direction
   - catalyst-adjusted movement
3. Turn relationships on in a usable form:
   - influence network
   - stakeholder dossier
   - relationship-weighted pressure
4. Turn session lifecycle on:
   - current phase overlay
   - phase-based risk modifiers
   - deadline pressure
5. Add the first policy-market primitives:
   - odds chart
   - volatility
   - support/resistance ladder
   - movement history

Must-have deliverable:

- the screen can show a credible implied-outcome view for a hearing, bill, or issue room using actual persistent infrastructure

## Build order inside the month

Priority order:

1. front-end flagship screen
2. committee-intel trust improvements
3. action conversion layer
4. missing DB infrastructure
5. real odds and relationship overlays

## What not to do in month 1

Do not:

- build too many extra pages
- pretend probabilities are robust before the model is real
- over-automate comms before evidence quality improves
- expand jurisdictions before Texas/Houston feel dominant

## North-star metric for the month

By the end of the month, a lobby firm user should be able to:

- open one screen
- see what committee or bill is moving
- understand why
- know who matters
- know what to do next
- generate a client-ready output

in under five minutes.
